## 🚨 发现的问题清单

### 问题 #1: 客户端 API 调用与后端路由不匹配
**严重性**: 🔴 高
**描述**: `client/lib/services/api_service.dart` 调用了以下后端不存在的接口：
- `POST /combat/calculate-damage` — 后端只有 `/combat/attack`
- `POST /combat/player-attack-mob` — 不存在
- `POST /combat/mob-attack-player` — 不存在
- `GET /combat/stats` — 不存在
- `GET /items` — 不存在（只有 `/inventory`）
- `GET /items/:id` — 不存在
- `GET /npcs/map/:mapId` — 不存在（只有 `/npcs/:id`）

**建议修复**: 在 `routes.go` 的 `registerGameRoutes` 下补充：
- 新 `Group("/combat")`：添加 `calculate-damage`/`player-attack-mob`/`mob-attack-player`/`stats`/`revive` 五个新 handler
- 新 `Group("/items")`：GET 物品列表/详情
- 修改 `registerNPCRoutes`：添加 `GET /map/:mapId` 获取某地图所有 NPC

### 问题 #2: WebSocket 客户端期望消息字段与服务端不一致
**严重性**: 🟠 中
**描述**: `WebSocketService` 发送的消息体包装在 `{"type", "payload", "sender_id", "room"}` 下，而服务端 `websocket_handler.go` 期望顶层字段 `type/channel/character_id/name/x/y/damage/...`

**建议修复**: 统一协议格式：
- **方案A（推荐）**: 修改服务端以解析 `payload` 结构（保持 Go WSMessage 结构体不变，增加 Payload 字段）
- **方案B**: 修改 Dart 端以匹配 Go 端（直接展开字段，无 `payload` 嵌套）

### 问题 #3: 资源实际文件缺失
**严重性**: 🟡 中
**描述**: `client/assets/` 下只有 README.md 清单，并无实际音频/图片/精灵文件
**建议修复**:
- 短期：在 `game_world.dart` 中使用程序化 Canvas 绘制占位（已有雏形）
- 中期：编写 `scripts/asset_extractor.go` 从 wz 或放置 CC0 资源
- 长期：在 `pubspec.yaml` 中声明 assets 路径并确保 Flutter 加载

### 问题 #4: AP/SP 自动分配（已确认修复 ✅）
**描述**: `game_service.go` 的升级流程中已包含 `AbilityPoint += 5` 和 `SkillPoint += 3`
**状态**: ✅ 已在 `LevelUp()` 和 `ProcessLevelUp()` 中确认实现

### 问题 #5: 职业差异化 HP/MP 增量（已确认修复 ✅）
**描述**: `game_service.go` 的 `getLevelUpHPMP()` 函数已实现职业差异化
**状态**: ✅ 已在 `constants.go` 的 `JobLevelUpStatsMap` 和 `game_service.go` 的 `getLevelUpHPMP()` 中确认实现
